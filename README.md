# Part 1: Relational Data & SQL Optimization

- ในไฟล์  Chicago Crime Data (2024 - Present) มีขนาดใหญ่เกินการดาวน์โหลดไฟล์ Raw Data โดยตรงมีขนาดใหญ่เกินไปสำหรับการจัดการบนหน่วยความจำ (RAM) ทั่วไป ผมเลยใช้เป็น API
- Data Ingestion
  - ผมได้ไปศึกษาวิธีการเปลี่ยนจาก API เป็น ไฟล์ CSV เนื่องจากผมถนัดในการใช้ CSV มากว่าผมได้ดึงมา 100k rows และดึงมาทีละ 5000 rows แล้วเอามาต่อกันจนครบ 100k rows
  - ผมได้ทำการใช้ time.sleep เพื่อ รับ ERROR 429
  - จากนั้นผมได้ศึกษาวิธีการ เอา CSV เข้า Mysql
  - โดยเริ่มจาก ผมแปลง Data type ของ date ก่อนเพื่อรองรับโจทย์ข้อต่อไป
  - จากที่ศึกษามาผมได้เลือกใช้ pymysql เพื่อใช้ python สื่อสารกับ mySql ได้ โดยแบ่งทีละ 5000 rows เพื่อนำเข้า Database 
- Schema Design
  - เริ่มแรกทำการเลือก cols ที่เหมาะกับ time-serie และ Geo คือ id','case_number','date','primary_type','district','latitude','longitude'
  - จากนั้นเริ่มทำการ Clean Data โดยผมทำการ clean Latitude ให้อยู่ระหว่าง -90 ถึง 90 และ Longitude ระหว่าง -180 **จุดนี้เป็นจุดที่มีปํญหาไม่สามารถนำเข้าใน Database เนื่องจากฐานข้อมูลมองเป็นเพียงตัวเลขทศนิยม 2 คอลัมน์แยกกันได้ผมจึงได้ศึกษาเพื่อเติมจนพบว่า มันต้องใช้ location POINT SRID 4326
  - ต่อมา Clean id ลบ id ที่ซ้ำกันออก
- Complex Querying
  - ผมได้ทำการหาตัวอย่างการทำ 7-Day Rolling Average เพื่อศึกษา ทำความเข้าใจ และพยายามเขียนตาม
    - เริ่มจาก หาวันล่าสุดที่เกิดคดี
    - นับจำนวนคดีแยกตามรายวันและรายเขต
    - ใช้ CROSS JOIN ระหว่างรายการวันที่และรายการเขตทั้งหมด
    - ใช้ COALESCE(count, 0) เพื่อเปลี่ยนค่า NULL ให้เป็น 0 ทำให้การคำนวณทางสถิติไม่ผิดเพี้ยน
    - ใช้ Window Function คำนวณค่าเฉลี่ยย้อนหลัง 7 วัน (6 วันก่อนหน้า + วันปัจจุบัน)
    - กรองข้อมูลให้เหลือเพียง 30 วันล่าสุด ตามโจทย์
  - Data Integrity
    - Missing latitude :  0.396 %
    - Missing longitude :  0.396 %
     - Data Handling Strategy
       - ใช้ Column ['block'] ส่งไปที่ API Google Map หรือ Nominatim เพื่อแปลงกลับมาเป็น Lat/lon
       - Check Bias ตรวจสอบก่อนว่าข้อมูล latitude กับ longitude ที่หายไปเป็นแบบสุ่มหรือแบบเฉพาะพท้นที่เพื่อไม่ให้ model เกิดการ biass
       - ทำการตรวจสอบพิกัดที่ไม่ได้อยู่ในขอบเขตของ chicago
       - ใช้ Uber H3 เพื่อแปลงพิกัดเป็น Discrete Area IDs เพื่อแก้ GPS Noise 
       - ทำข้อมูล 2 ชุด ชุดแรกเป็น original data ชุดสองเป็น ข้อมูลที่เราจัดการมา เพื่อดูว่าข้อมูลที่เราทำมา สร้าง Noise หรือ ทำให้ Model เกิด Bias ไปจากความจริงไหม
